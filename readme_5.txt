1. Порівняйте та назвіть переваги та недоліки таких методів сортування: сортування бульбашкою,
сортування вибором, сортування вставкою.
1.Переваги та недоліки методів сортування:
    •Сортування бульбашкою:
        Переваги: Простий для реалізації та розуміння.
        Недоліки: Повільний, має O(n²) у найгіршому випадку, навіть для майже відсортованих масивів.
    •Сортування вибором:
        Переваги: Менше переміщень порівняно з бульбашкою, не потребує багато обмінів.
        Недоліки: Повільний (O(n²)) і не є стабільним.
    •Сортування вставкою:
        Переваги: Ефективний для майже відсортованих масивів, стабільний, швидкий у кращому випадку O(n).
        Недоліки: Не підходить для великих не відсортованих масивів (O(n²)).
2. Оцініть кожен з методів відповідно до наступних критеріїв:
- час роботи;
- потреби у додатковій пам’яті;
- стабільність.
2.Оцінка за критеріями:

    •Час роботи:
        Бульбашка: O(n²)
        Вибір: O(n²)
        Вставка: O(n²), але O(n) для частково відсортованих масивів.
    •Пам'ять: Усі три алгоритми потребують O(1) додаткової пам’яті.
    •Стабільність:
        Бульбашка: Стабільний.
        Вибір: Не стабільний.
        Вставка: Стабільний.
 3. Який випадок є найкращім або найгіршим для роботи цих алгоритмів? (Наприклад: частково відсортований масив або масив, відсортований у зворотному порядку). Який з методів
забезпечить у цих випадках найкращій/найгірший результат?
3.Найкращий/найгірший випадок:

    •Бульбашка: Найгірший для масивів, відсортованих у зворотному порядку. Найкращий — повністю відсортований масив.
    •Вибір: Незалежний від початкового стану масиву.
    •Вставка: Найкращий для майже відсортованих масивів. Найгірший — зворотний порядок.
4. Що означають позначення O(1), O(n), O(n²)?
4. •O(1): Константний час, не залежить від розміру даних.
   •O(n): Лінійний час, пропорційний розміру даних.
   •O(n²): Квадратичний час, росте швидко зі збільшенням розміру масиву.
